# 10_entities.md — Создание сущностей (entities)

## Где создавать

Новую сущность создаем в `packages/entities/[entity-name]/`

```
*/entities/
├── [entity-name]/
│ ├── api.ts    # Запросы к API
│ ├── type.ts   # TypeScript типы и интерфейсы
│ ├── utils.ts  # Хелперы для работы с сущностью
│ └── index.ts  # Публичное API (экспорты)(только начали внедрять)
```

## Когда создавать сущность

Сущность — это бизнес-объект, который:
- Имеет свою модель данных в БД
- Используется в нескольких местах приложения
- Имеет свои API эндпоинты

**Примеры:** User, Order, Hardware, Object, Document, Incident

## Правила создания

1. **Не дублировать** — проверить существующие сущности
2. **Не смешивать** — одна сущность = одна папка
3. **Типизировать всё** — каждый API ответ должен иметь тип
4. **Экспортировать только нужное** — в `index.ts` только публичное API
